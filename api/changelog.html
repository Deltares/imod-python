Changelog#

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

[1.0.0rc7] - 2025-10-28#

Added#

  • imod.mf6.Modflow6Simulation.set_validation_settings() to set validation settings for a MODFLOW 6 simulation. See imod.mf6.ValidationSettings for more information.

Changed#

  • No automatic validation upon calling imod.mf6.Modflow6Simulation.regrid_like() anymore. Use the validate argument of imod.mf6.Modflow6Simulation.write() to validate the regridded model upon writing instead.

  • imod.mf6.River now ignore confined cells (icelltype == 0) when validating whether the river bottom elevation is below the model bottom elevation.

  • Moved imod.select.get_upper_active_layer_number(), imod.select.get_upper_active_cells(), imod.select.get_lower_active_cells(), and imod.select.get_lower_active_layer_number() from imod.prepare. to imod.select.

  • imod.mf6.Dispersion now is not a required package for imod.mf6.GroundwaterTransportModel anymore.

  • No validation anymore for icelltype upon writing imod.mf6.SpecificStorage and imod.mf6.StorageCoefficient.

Removed#

  • Removed imod.select.upper_active_layer function, use imod.select.get_upper_active_layer_number() instead.

Fixed#

  • Fixed bug where imod.mf6.Modflow6Simulation.split() could result in empty exchanges being present in the split_exchanges package list, when two models were isolated by inactive cells from each other. These empty exchanges are now removed.

  • Fixed bug where imod.mf6.Modflow6Simulation.split(), imod.mf6.Modflow6Simulation.regrid_like(), and imod.mf6.Modflow6Simulation.clip_box() would not copy imod.mf6.ValidationSettings.

  • landuse, soil_physical_unit, active for imod.msw.GridData are now properly regridded with the mode statistic when using imod.msw.GridData.regrid_like().

[1.0.0rc6] - 2025-08-28#

Small post-release to fix rendering of documentation online.

[1.0.0rc5] - 2025-08-27#

Added#

  • imod.mf6.River.reallocate(), imod.mf6.Drainage.reallocate(), imod.mf6.GeneralHeadBoundary.reallocate(), imod.mf6.Recharge.reallocate() to reallocate the package data to a new discretization or imod.mf6.NodePropertyFlow package, or to use a different imod.prepare.ALLOCATION_OPTION or imod.prepare.DISTRIBUTING_OPTION.

  • Added imod.mf6.HorizontalFlowBarrierResistance.snap_to_grid() and imod.mf6.SingleLayerHorizontalFlowBarrierResistance.snap_to_grid() to debug how horizontal flow barriers are snapped to a grid.

  • Added imod.mf6.Modflow6Simulation.create_partition_labels() to create partition labels for a MODFLOW 6 simulation from its idomain. This is useful for splitting a simulation into multiple submodels.

  • imod.mf6.AdaptiveTimeStepping to specify adaptive time stepping settings for MODFLOW 6 simulations.

  • The ats_percel argument to imod.mf6.AdvectionTVD, imod.mf6.AdvectionUpstream, imod.mf6.AdvectionCentral to adapt the time step based on the maximum fraction of a cell that a solute parcel is allowed to travel.

Fixed#

  • Reduce noisy warnings in models loaded with imod.mf6.Modflow6Simulation.from_imod5_data() which have layers with cells with zero thicknesses.

  • Issue where regridding lead to excessively large inactive areas.

  • Issue where regridding would lead to very large negative integer values (like IDOMAIN) for inactive areas.

  • Issue where imod.mf6.Well.from_imod5_data() and imod.mf6.LayeredWell.from_imod5_data() would throw a KeyError 0 upon trying to resample timeseries with a non-zero index.

  • Fixed bug where imod.mf6.HorizontalFlowBarrierResistance, imod.mf6.SingleLayerHorizontalFlowBarrierResistance and other HFB packages would have resistances that were double the expected value with xugrid >= 0.14.2

  • The states_for_boundary argument now also works for tranport models in imod.mf6.Modflow6Simulation.clip_box().

  • Fix bug where imod.mf6.Modflow6Simulation.clip_box() and the states_for_boundary argument would place these bc at the incorrect places with unstructured grids.

  • Fixed bug where imod.mf6.SourceSinkMixing.from_flow_model() would return an error upon adding a package which cannot have a concentration, such as imod.mf6.HorizontalFlowBarrierResistance.

  • Broken names for outer_csvfile and inner_csvfile in the imod.mf6.Solution MODFLOW 6 template file.

Changed#

  • imod.mf6.StructuredDiscretization.from_imod5_data() and imod.mf6.NodePropertyFlow.from_imod5_data() now automatically load the dataset into memory. This improves performance when loading models with multiple topsystem packages.

  • No upper limit anymore for mod_id in mod2svat.inp for imod.msw.CouplerMapping.

  • imod.prepare.create_partition_labels() now takes an idomain grid instead of imod.mf6.Modflow6Simulation as first argument. To generate partition labels from a imod.mf6.Modflow6Simulation straightaway, use the newly added imod.mf6.Modflow6Simulation.create_partition_labels() method instead.

Removed#

  • Removed imod.mf6.WellDisStructured and imod.mf6.WellDisVertices. Use imod.mf6.Well and imod.mf6.LayeredWell instead. The imod.mf6.Well package can be used to specify wells with filters, imod.mf6.LayeredWell directly to layers.

  • Removed imod.mf6.multimodel.partition_generator.get_label_array, use imod.prepare.create_partition_labels() instead.

  • Removed imod.formats.idf.read use imod.formats.idf.open() instead.

  • Removed imod.formats.rasterio.read use imod.formats.rasterio.open() instead.

  • Removed head argument for imod.mf6.InitialConditions, use start instead.

  • Removed cell_averaging argument for imod.mf6.NodePropertyFlow, use alternative_cell_averaging instead.

  • Removed set_repeat_stress method from boundary condition packages like imod.mf6.River. Use repeat_stress argument instead.

  • Removed time_discretization method from imod.mf6.Modflow6Simulation and imod.wq.SeawatModel. Use imod.mf6.Modflow6Simulation.create_time_discretization() and imod.wq.SeawatModel.create_time_discretization() instead.

  • Removed imod.util.round_extent, use imod.prepare.round_extent() instead.

  • Removed imod.prepare.Regridder. Use the xugrid regridder instead.

[1.0.0rc4] - 2025-06-20#

Added#

  • Added weights argument to imod.prepare.create_partition_labels() to weigh how the simulation should be partioned. Areas with higher weights will result in smaller partions.

  • iMOD Python version is now written in a comment line to MODFLOW6 and MetaSWAP’s para_sim.inp files. This is useful for debugging purposes.

  • Added option ignore_time_purge_empty to imod.mf6.Modflow6Simulation.split() to consider a package empty if its first times step is all nodata. This can save a lot of time splitting transient models.

  • Add imod.mf6.ValidationSettings to specify validation settings for MODFLOW 6 simulations. You can provide it to the imod.mf6.Modflow6Simulation constructor.

Fixed#

  • Upon providing an unexpected coordinate in the mask or regridding grid, imod.mf6.Modflow6Simulation.regrid_like() and imod.mf6.Modflow6Simulation.mask_all_models() now present the unexpected coordinates in the error message.

  • imod.mf6.VerticesDiscretization now correctly sets the xorigins and yorigins options in the .disv file. Incorrect origins cause issues when splitting models and computing with XT3D on the exchanges.

  • imod.mf6.open_cbc() and imod.mf6.open_hds() now account for xorigins and yorigins for models ran with imod.mf6.VerticesDiscretization. WARNING: Given that these were set incorrectly in previous versions of iMOD Python (see previous item in this list), this means that reading MODFLOW6 DISV output of models generated with a previous version of iMOD Python will result in a grid with an erroneous offset. You can work around this by creating the model again with this version of iMOD Python or newer.

  • imod.mf6.Modflow6Simulation.split() supports label array with a different name than "idomain".

  • imod.msw.MetaSwapModel.from_imod5_data() now supports the usage of relative paths for the extra files block.

  • Bug in imod.msw.Sprinkling.write() where MetaSWAP svats with surface water sprinkling and no groundwater sprinkling activated were not written to scap_svat.inp.

  • imod.msw.IdfMapping swapped order of y_grid and x_grid in dictionary for writing the correct order of coordinates in idf_svat.inp.

  • Improved performance of imod.mf6.Modflow6Simulation.split() and imod.mf6.Modflow6Simulation.mask_all_models() when using dask.

  • Fixed bug in imod.mf6.Modflow6Simulation.mask_all_models() for unstructured grids with a spatial dimension that differs from the default "mesh2d_nFaces".

  • Fixed bug in imod.mf6.Well.cleanup() and imod.mf6.LayeredWell.cleanup() which caused an error when called with an unstructured discretization.

  • Fixed bug in imod.formats.prj.open_projectfile_data() which caused an error when a periods keyword was used having an upper case.

  • Poor performance of imod.mf6.Well.from_imod5_data() and imod.mf6.LayeredWell.from_imod5_data() when the imod5_data contained a well system with a large number of wells (>10k).

  • imod.mf6.River.from_imod5_data(), imod.mf6.Drainage.from_imod5_data(), imod.mf6.GeneralHeadBoundary.from_imod5_data() can now deal with constant values for variables. One variable per package still needs to be a grid.

  • Fix bug where an error was thrown in get_non_grid_data when calling the .cleanup and regrid_like methods on a boundary condition package with a repeated stress. For example, imod.mf6.River.cleanup() or imod.mf6.River.regrid_like().

  • Fix bug where an error was thrown in imod.mf6.Well when an entry had to be filtered and its id didn’t match the index.

  • Improved performance of imod.mf6.Modflow6Simulation.split for structured models, as unnecessary masking is avoided.

  • Fixed warning thrown by type dispatcher about ~GeoDataFrameType

  • Fixed bug where variables in a package with only a "layer" coordinate could not be regridded or masked.

Changed#

  • imod.wq.SeawatModel.write() now throws an error if trying to write in a directory with a space in the path. (iMOD-WQ does not support this.)

  • imod.mf6.multimodel.partition_generator.get_label_array moved to imod.prepare.create_partition_labels().

  • imod.prepare.create_partition_labels() structured grids are now partioned by METIS instead (just like already was the case for unstructured grids). This results in more balanced partitions for grids with non-square domains or lots of inactive cells. Downside is that the partitions are more often than not perfectly rectangular in shape.

  • imod.prepare.create_partition_labels() now returns a griddata with the name "label" instead of "idomain".

  • Upon providing the wrong type to one of the options of imod.mf6.GroundwaterFlowModel, imod.mf6.GroundwaterTransportModel, this will throw a ValidationError upon initialization and writing.

  • You can now also provide repeat_stress as dictionary to imod.mf6 boundary conditions, such as imod.mf6.River, imod.mf6.Drainage, and imod.mf6.GeneralHeadBoundary.

  • imod.mf6.ConstantHead.from_imod5_data(), imod.mf6.GeneralHeadBoundary.from_imod5_data(), imod.mf6.River.from_imod5_data(), imod.mf6.Recharge.from_imod5_data(), and imod.mf6.Drainage.from_imod5_data() now forward fill data over time, instead of clipping, when selecting a start time that is inbetween two data records.

  • imod.mf6.ConstantHead.from_imod5_data() and imod.mf6.Recharge.from_imod5_data() got extra arguments for period_data, time_min and time_max.

  • imod.visualize.read_imod_legend() now also returns the labels as an extra argument. Update your code by changing colors, levels = read_imod_legend(...) to colors, levels, labels = read_imod_legend(...).

[1.0.0rc2] - 2025-03-05#

From this release on, we recommend using xugrid’s regridding utilities for regridding individual grids instead of imod.prepare.Regridder. Xugrid’s regridders are tested to be about 10 times faster than imod.prepare.Regridder. There is one small difference: xugrid’s xugrid.BaryCentricInterpolator considers sample points of the destination grid that lie on the source grid’s cell edges to be inside, whereas imod.prepare.Regridder considers them to be outside. This difference is negligible for most applications, but might create slightly fewer np.nan values than before.

Removed#

  • imod.flow module has been removed for generating iMODFLOW models. Use imod.mf6 instead to generate MODFLOW 6 models.

Added#

  • Support for Python 3.13.

  • imod.mf6.Recharge.from_imod5_data(), imod.mf6.River.from_imod5_data(), imod.mf6.Drainage.from_imod5_data(), and imod.mf6.GeneralHeadBoundary.from_imod5_data() now assign negative layer numbers to the first active layer.

  • imod.prepare.DISTRIBUTING_OPTION() got a new setting by_corrected_thickness. This matches DISTRCOND=-1 in iMOD5.

  • imod.prepare.cleanup_hfb() to clean up HFB geometries.

  • imod.mf6.HorizontalFlowBarrierResistance.cleanup(), imod.mf6.SingleLayerHorizontalFlowBarrierResistance.cleanup(), to clean up HFB geometries crossing inactive model cells.

  • imod.util.RegridderWeightsCache to store regridder weights for regridding multiple times.

  • imod.util.RegridderType to specify regridder types.

Changed#

  • imod.formats.prj.open_projectfile_data() now also assigns negative and zero layer numbers to grid coordinates.

  • In imod.mf6.StructuredDiscretization, IDOMAIN can now respectively be > 0 to indicate an active cell and <0 to indicate a vertical passthrough cell, consistent with MODFLOW 6. Previously this could only be indicated with 1 and -1.

  • imod.mf6.Well.from_imod5_data() and imod.mf6.LayeredWell.from_imod5_data() now also accept the argument times = "steady-state", for the simulation is assumed to be “steady-state” and well timeseries are averaged.

  • The drn attribute of imod.prepare.SimulationAllocationOptions has the at_elevation of imod.prepare.ALLOCATION_OPTION() option now set as default. This means by default drainage cells are placed differently in imod.mf6.Modflow6Simulation.from_imod5_data().

  • imod.mf6.Well, imod.mf6.LayeredWell, imod.prepare.assign_wells(), imod.mf6.Well.from_imod5_data() and imod.mf6.LayeredWell.from_imod5_data() now have default values for minimum_thickness and minimum_k set to 0.0.

  • When intitating a MODFLOW 6 package with a layer coordinate with values <= 0, iMOD Python will throw an error.

  • imod.mf6.HorizontalFlowBarrierResistance, imod.mf6.SingleLayerHorizontalFlowBarrierResistance and other HFB now validate whether proper type of geometry is provided, respectively Polygon for imod.mf6.HorizontalFlowBarrierResistance, and LineString for imod.mf6.SingleLayerHorizontalFlowBarrierResistance.

  • Relaxed validation for imod.msw.MetaSwapModel if FileCopier package is present.

  • Change aterisk to dash and tabs to four spaces in ValidationError messages.

  • imod.prepare.laplace_interpolate() has been simplified, using scipy.sparse.linalg.cg as the backend. We’ve remove the support for the ibound argument, the iter1 argument has been dropped, mxiter has been renamed to maxiter, close has been renamed to rtol.

  • Moved imod.mf6.utilities.regrid.RegridderWeightsCache to the imod.util.regrid.RegridderWeightsCache.

[0.6.0] - 2019-03-15#

The primary change is that a number of functions have been renamed to better communicate what they do.

The load function name was not appropriate for IDFs, since the IDFs are not loaded into memory. Rather, they are opened and the headers are read; the data is only loaded when needed, in accordance with xarray’s design; compare for example xarray.open_dataset. The function has been renamed to open.

Similarly, load for IPFs has been deprecated. imod.ipf.read now reads both single and multiple IPF files into a single pandas.DataFrame.

Removed#

  • imod.idf.setnodataheader

Deprecated#

  • Opening IDFs with imod.idf.load, use imod.idf.open instead

  • Opening a set of IDFs with imod.idf.loadset, use imod.idf.open_dataset instead

  • Reading IPFs with imod.ipf.load, use imod.ipf.read

  • Reading IDF data into a dask array with imod.idf.dask, use imod.idf._dask instead

  • Reading an iMOD-seawat .tec file, use imod.tec.read instead.

Changed#

  • Use np.datetime64 when dates are within time bounds, use cftime.DatetimeProlepticGregorian when they are not (matches xarray defaults)

  • assert is no longer used to catch faulty input arguments, appropriate exceptions are raised instead

Fixed#

  • idf.open: sorts both paths and headers consistently so data does not end up mixed up in the DataArray

  • idf.open: Return an xarray.CFTimeIndex rather than an array of cftime.DatimeProlepticGregorian objects

  • idf.save properly forwards nodata argument to write

  • idf.write coerces coordinates to floats before writing

  • ipf.read: Significant performance increase for reading IPF timeseries by specifying the datetime format

  • ipf.write no longer writes ,, for missing data (which iMOD does not accept)

[0.5.0] - 2019-02-26#

Removed#

  • Reading IDFs with the chunks option

Deprecated#

  • Reading IDFs with the memmap option

  • imod.idf.dataarray, use imod.idf.load instead

Changed#

  • Reading IDFs gives delayed objects, which are only read on demand by dask

  • IDF: instead of res and transform attributes, use dx and dy coordinates (0D or 1D)

  • Use cftime.DatetimeProlepticGregorian to support time instead of np.datetime64, allowing longer timespans

  • Repository moved from https://gitlab.com/deltares/ to https://gitlab.com/deltares/imod/

Added#

  • Notebook in examples folder for synthetic model example

  • Support for nonequidistant IDF files, by adding dx and dy coordinates

Fixed#

  • IPF support implicit itype
